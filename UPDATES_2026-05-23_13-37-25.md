# Astrometric Scale Engine - Developer Updates Log
**Date:** 2026-05-23 13:37:25 PDT

This log documents the specific architectural and file-level modifications made during the Interstellar UI and Data Overhaul to assist future maintainers.

## 1. UI & State Architecture Refinements
- **Hull Structural Mass Slider:** 
  - **File:** `src/components/FlightConfig.tsx`
  - **Change:** Replaced the static numeric input for `mass` with an `<input type="range">` mapped from 25 tonnes to 10M tonnes (step 25). 
  - **Logic:** Implemented a ternary threshold system inside the component to render dynamic classifications (e.g., `Scout` for <= 75t, `Generation Ship` for >= 10Mt) beneath the slider.
- **Time Header Alignment:**
  - **File:** `src/components/FlightDashboard.tsx` (Assumed via UI visual updates) / Component Grid
  - **Change:** Restructured the DOM grid output to side-by-side header formats, preventing line-wrap issues and vertically aligning `Arrival (Earth Time)` with `Arrival (Ship Time)`.
- **CSS Rendering Fix for Native Form Elements:**
  - **File:** `src/index.css`
  - **Change:** Modified global stylesheet rules to explicitly select `select`, `optgroup`, `option`, and `input[type="text"]`.
  - **Reasoning:** Natively, some browsers force a white background on these elements. By explicitly assigning `background-color: var(--color-bg-base)` and `color: var(--color-text-primary)`, we prevent white-on-white text clipping. Inline styling overrides were concurrently stripped from `FlightConfig.tsx`.

## 2. Data Pipeline: HYG Interstellar Registry
- **Deprecation of Dynamic Gaia API:**
  - **File:** `scripts/build_data.py`
  - **Change:** The `astroquery.gaia` integration was stripped. It generated randomized, unreadable tracking codes (e.g., "Gaia DR3 44728...") which harmed user experience.
- **HYG Database Migration:**
  - **Implementation:** `build_data.py` now leverages `urllib.request` to dynamically download the `hygdata_v3.csv` from Codeberg/Github (`https://raw.githubusercontent.com/astronexus/HYG-Database...`).
  - **Parsing Logic:** The Python script parses the CSV, strictly filtering for stars with a `proper` name, OR magnitude `< 5.0` that have a `bayer`/`flam` designation mapped to a constellation (`con`). This scales the dataset to **1,430 Milky Way objects**.
- **Sol Anchor & Custom Targets:**
  - Sol was explicitly mapped to `[0.0, 0.0, 0.0]` cartesian coordinates.
  - A `custom_targets` array in `build_data.py` was retained to inject extra-galactic elements (Messier Catalog), Exoplanets, and spacecraft (Voyager 1).
- **TypeScript Schema Modification:**
  - **File:** `src/state/useFlightStore.ts`
  - **Change:** The `AstroObject` interface was expanded.
  - **New Fields:** Added `catalog_origin?: string` and nested `display_metrics?: { constellation?: string, apparent_magnitude?: number, true_distance_ly?: number, scale_narrative?: string }`.

## 3. Advanced Filtering Mechanisms
- **Flight Configuration Component Upgrade:**
  - **File:** `src/components/FlightConfig.tsx`
  - **Change:** Introduced `originFilter`, `destinationFilter`, `originCategory`, and `destinationCategory` state variables.
- **Categorized Optgroups:**
  - `renderOptions(filterStr, categoryStr)` now groups the `activeCatalog` mapping by `catalog_origin` (e.g., `<optgroup label="Messier Catalog">`).
- **Native Real-Time Search:**
  - Injected an `<input type="text">` and `<select>` (for category toggling) above the origin/destination selectors.
  - **Filter Logic:** Strings are lower-cased and evaluated against `obj.name`, `obj.catalog_origin`, and `obj.display_metrics.constellation`. Empty optgroups automatically collapse from the DOM, allowing high-performance React filtering without relying on bloated libraries like `react-select`.
- **Dynamic HUD Mount:**
  - Created `renderReadout(node: AstroObject)` to mount an informational pane below the dropdown containing the `true_distance_ly` and `scale_narrative` whenever a node is selected.
